## Goal

Introduce the data model and export behavior for externally defined TikZ styles.

Imported styles should be referenced by key in generated commands, but their full `\tikzset{...}` definitions should **not** be inlined into generated TikZ.

Generated TikZ should only include comments instructing the user to load the relevant style file externally.

Example desired output:

```tex
% External TikZ styles referenced below.
% Load before this picture, for example:
%   \input{mygeometry.sty}
% or include the package/style file in your LaTeX preamble.
\begin{tikzpicture}[...]
  \draw[3cat/phys/1strata/color/x] ...
\end{tikzpicture}
```

The comments are instructions only. Do not emit an active `\input{...}` command unless the user explicitly enables such behavior later.

## Data model

Add model for external style sources and imported style references.

Suggested:

```ts
type ExternalTikzStyleSource = {
  id: string;
  name: string;          // e.g. "mygeometry.sty"
  loadHint: string;      // e.g. "\\input{mygeometry.sty}"
};

type ImportedTikzStyleReference = {
  id: string;
  key: string;           // e.g. "3cat/phys/1strata/color/x"
  sourceId: string;
  displayName: string;
  targets: TikzStyleTarget[];
};
```

Suggested target enum:

```ts
type TikzStyleTarget =
  | "draw"
  | "filldraw"
  | "node"
  | "curve"
  | "sheet"
  | "point"
  | "label"
  | "region";
```

Exact shape can differ.

Requirements:

- style source metadata saves/loads;
- imported style keys save/load;
- old diagrams still load;
- keys are treated as TikZ option keys;
- keys are not parsed as geometry;
- invalid blank keys rejected;
- source names sanitized for display but not over-sanitized for comments.

## Applying imported style references

Allow an element/preset to carry a custom TikZ style key reference.

Examples:

Curve:

```tex
\draw[3cat/phys/1strata/color/x] ...
```

Sheet/region:

```tex
\filldraw[3cat/phys/1strata/color/y] ...
```

Point/label:

```tex
\node[3cat/phys/3strata/shape/L] at (...) {};
```

Requirements:

- structured style still drives SVG preview;
- imported style key is added to TikZ command options;
- unknown/imported style keys are not expanded;
- no `\tikzset` definition is generated;
- no active `\input` is generated by default;
- output includes comments listing the needed external style sources.

If an element has both structured style and imported style key:

- output both in a deterministic order;
- choose whether imported style comes before or after structured options;
- document/test the order.

## TikZ output comments

Add comments near the top of generated TikZ, before `\begin{tikzpicture}`, such as:

```tex
% External TikZ styles referenced below.
% Load these files in your LaTeX preamble or before the picture:
% - mygeometry.sty
% Suggested:
%   \input{mygeometry.sty}
```

Do not emit the entire `\tikzset`.

Do not emit active `\input`.

If multiple external sources are used, list all of them once.

If no imported styles are used, emit no external-style comment.
