## Goal

Add a toolbar Variable Manager where users can define symbolic variables.

Variables should export to TikZ as `\pgfmathsetmacro`.

Example:

User defines:

```text
R = 2
q = 30
```

Generated TikZ should include:

```tex
\pgfmathsetmacro{\R}{2}
\pgfmathsetmacro{\q}{30}
```

These variables will be used by symbolic coordinate input in later phases.

## Variable model

Suggested:

```ts
type SymbolicVariable = {
  id: string;
  name: string;       // user name, e.g. "R"
  macroName: string;  // TeX macro name, e.g. "R"
  expression: string; // expression source, e.g. "2" or "R0/2"
  previewValue: number;
};
```

Requirements:

- variables are diagram/export data if they affect generated TikZ;
- save/load round-trip;
- old diagrams without variables still load;
- duplicate variable names rejected;
- duplicate macro names rejected;
- blank names rejected;
- invalid expressions rejected;
- dependency cycles rejected if variables can depend on variables.

## Macro name policy

User example expects:

```text
R -> \R
q -> \q
```

Implement a clear TeX macro policy.

Preferred MVP:

- variable names must match:

```text
[A-Za-z][A-Za-z0-9_]*
```

- macro names are generated by stripping unsupported characters or by using the same name when TeX-safe.
- If TeX control sequence names with digits/underscore are problematic, restrict macro names to letters only and reject others.

A conservative policy is acceptable:

```text
[A-Za-z]+
```

If using only letters, document that variables like `R1` are not allowed yet.

Do not allow macro names that begin with `\`.

Do not allow raw TeX in variable names.

## Variable expression values

Variable values can be numeric literals or supported symbolic expressions from Phase 19A.

Examples:

```text

```

If dependencies are supported:

- evaluate variables in dependency order;
- reject cycles;
- reject unknown variables.

If dependencies are not supported in MVP:

- require variable expressions to be numeric only;

Preferred: support dependency DAG.

## TikZ export placement

Export variables as `\pgfmathsetmacro`.

### Standalone mode

Variables may be emitted before `\begin{tikzpicture}` according to current standalone setup conventions.

### Inline math mode

Variables must be emitted inside `tikzpicture` near the top, with no blank lines.

Example:

```tex
\begin{tikzpicture}[baseline={([yshift=-.5ex]current bounding box.center)}]
    %----------------------------------------
    % Variables
    %----------------------------------------
    \pgfmathsetmacro{\R}{2}
    \pgfmathsetmacro{\q}{30}
    ...
\end{tikzpicture}
```

Inline output must still contain no blank lines.
